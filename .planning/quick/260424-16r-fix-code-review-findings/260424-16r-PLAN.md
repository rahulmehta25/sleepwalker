  <action>
In `dashboard/lib/queue.ts`, import `lockfile` from `proper-lockfile` at the top:

```ts
import lockfile from "proper-lockfile";
```

Rewrite `updateLocalStatus` to acquire an exclusive lock on the queue file before
read-modify-write and release it in a finally block. Use the `lockfile.lock` /
`lockfile.unlock` API (not the callback form). Stale lock threshold: 10_000 ms
(the `proper-lockfile` default), retries: 3, stale: 10000. The lock target is the
queue file path (the JSONL itself, not a sidecar — proper-lockfile creates a `.lock`
companion automatically).

Pattern to follow:
```ts
export function updateLocalStatus(id: string, status: QueueStatus): boolean {
  const f = ensureFile();
  let release: (() => Promise<void>) | undefined;
  try {
    release = lockfile.lockSync(f, { retries: 3, stale: 10_000 });
    const entries = parseLines(fs.readFileSync(f, "utf8"));
    const idx = entries.findIndex((e) => e.id === id);
    if (idx === -1) return false;
    entries[idx].status = status;
    fs.writeFileSync(f, entries.map((e) => JSON.stringify(e)).join("\n") + "\n");
    return true;
  } finally {
    release?.();
  }
}
```

Note: `lockfile.lockSync` returns a release function. Import type from
`@types/proper-lockfile` which is already in devDependencies.

Do NOT use the async form — the existing function is synchronous and callers depend
on a boolean return value without await. Keep it synchronous.

Also add a test file `dashboard/tests/queue-lock.test.ts` using `makeTempHome()`
isolation (see CONVENTIONS.md pattern — set `process.env.HOME` to a temp dir
so the real `~/.sleepwalker/queue.jsonl` is never touched). Write two entries,
call `updateLocalStatus` twice back-to-back, assert both updated correctly and
