```bash
# --- Helper: audit emit (QUEU-04: flock-wrapped to eliminate concurrent-write race) ---
# Per v0.1 CONCERNS.md §concurrent JSONL race + RESEARCH §1.7 live synthetic.
# Uses POSIX flock(1) FD form on ${LOCK_FILE} sidecar with 5s timeout.
# flock missing at runtime => nonzero exit from the subshell => supervisor
# continues (graceful degradation per RESEARCH §1.6 "option 2" for supervisor).
audit_emit() {
  local ts event extra
  ts="$(date -u +%Y-%m-%dT%H:%M:%SZ)"
  event="$1"
  extra="${2:-}"
  if [ -n "$extra" ]; then
    (
      flock -w 5 -x 200 || true
      printf '{"ts":"%s","fleet":"%s","runtime":"%s","event":"%s",%s}\n' \
        "$ts" "$FLEET" "$RUNTIME" "$event" "$extra" >> "$AUDIT_FILE"
    ) 200>"$LOCK_FILE"
  else
    (
      flock -w 5 -x 200 || true
      printf '{"ts":"%s","fleet":"%s","runtime":"%s","event":"%s"}\n' \
        "$ts" "$FLEET" "$RUNTIME" "$event" >> "$AUDIT_FILE"
    ) 200>"$LOCK_FILE"
  fi
}
```

**Idiom choice (per PATTERNS.md §Writer 2):** FD-form `( flock -x -w 5 200; printf ... >> "$AUDIT_FILE" ) 200>"$LOCK_FILE"`.
Rationale:
- No nested-quoting hazard (command form requires passing the printf as a string to `sh -c`, which breaks with the `%s` formatters and variable expansion)
- Block can contain multiple statements under one lock if future logging grows
- Matches util-linux idiom familiar to anyone reading flock(1) man page
- RESEARCH §1.3 verified FD-form produces identical 0% corruption as command form

**Failure-mode choice (per RESEARCH §1.6 supervisor path):** `flock ... || true` — if flock times out OR is missing, fall through to unlocked append. Preserves the audit entry (critical path for the supervisor — the audit is the only record of the run). Re-introduces the v0.1 race for that one entry but prevents catastrophic data loss. Hook writer (Plan 05-05) uses a stricter failure path because the hook can tolerate dropped entries.
