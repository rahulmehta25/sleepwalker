This plan's amendment is a new prereq check using the exact idiom v0.1's jq check already uses. Running the amended install.sh on a fresh Mac without flock now fails loud before touching any state — preferable to the alternative of the supervisor+hook silently falling through to unlocked writes. The phase_context in the plan request explicitly acknowledges this is a legitimate v0.2 extension documented at exit gate time.

Purpose: Plans 05-04 + 05-05 flock-wrap both audit writers. Without flock on PATH, the supervisor degrades gracefully (unlocked-fallback per RESEARCH §1.6) but the hook fails strictly (drops audit entries). Making flock a hard prereq at install time gives users a clear, actionable error up front — the pattern v0.1 already uses for jq.
