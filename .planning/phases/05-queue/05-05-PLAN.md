```bash
#!/bin/bash
# Sleepwalker PostToolUse hook: append every tool call from a sleepwalker
# session to the audit log.

set -euo pipefail

AUDIT_FILE="${HOME}/.sleepwalker/audit.jsonl"
HOOK_DIR="$(cd "$(dirname "$0")" && pwd)"

mkdir -p "${HOME}/.sleepwalker"
touch "$AUDIT_FILE"

INPUT="$(cat)"
SESSION_ID=$(echo "$INPUT" | jq -r '.session_id // ""')
TRANSCRIPT=$(echo "$INPUT" | jq -r '.transcript_path // ""')

FLEET="${SLEEPWALKER_FLEET:-}"
if [ -z "$FLEET" ]; then
  FLEET=$("$HOOK_DIR/_detect_fleet.sh" "$SESSION_ID" "$TRANSCRIPT" 2>/dev/null || echo "")
fi

if [ -z "$FLEET" ]; then
  printf '{}\n'
  exit 0
fi

TS=$(date -u +%Y-%m-%dT%H:%M:%SZ)
TOOL_NAME=$(echo "$INPUT" | jq -r '.tool_name // ""')
TOOL_INPUT_JSON=$(echo "$INPUT" | jq -c '.tool_input // {}')
OUTPUT_PREVIEW=$(echo "$INPUT" | jq -r '.tool_response // .tool_output // ""' | head -c 500)
OUTPUT_LEN=$(echo "$INPUT" | jq -r '.tool_response // .tool_output // ""' | wc -c | tr -d ' ')

ENTRY=$(jq -nc \
  --arg ts "$TS" \
  --arg fleet "$FLEET" \
  --arg session "$SESSION_ID" \
  --arg tool "$TOOL_NAME" \
  --argjson input "$TOOL_INPUT_JSON" \
  --arg out_preview "$OUTPUT_PREVIEW" \
  --argjson out_len "$OUTPUT_LEN" \
  '{ts:$ts,fleet:$fleet,session:$session,tool:$tool,input:$input,output_preview:$out_preview,output_length:$out_len}')

echo "$ENTRY" >> "$AUDIT_FILE"

printf '{}\n'
```

Target (amended):

```bash
HOOK_DIR="$(cd "$(dirname "$0")" && pwd)"

# ... existing jq construction unchanged ...

# QUEU-04: flock-wrap the append to serialize against bin/sleepwalker-run-cli's
# audit_emit. Per RESEARCH §1.6 the hook uses strict failure (no unlocked
# fallback) — a dropped audit line is recoverable on the next PostToolUse call;
# a race-corrupted line is not. Shared lock file ${HOME}/.sleepwalker/audit.jsonl.lock
# is the SAME sidecar the supervisor locks — one mutex, two writers.
(
  flock -w 5 -x 200
  echo "$ENTRY" >> "$AUDIT_FILE"
) 200>"$LOCK_FILE"

printf '{}\n'
```

Key difference from Plan 05-04 (supervisor): NO `|| true` on the flock call. The hook's `set -euo pipefail` (line 7) means a nonzero flock exit propagates as a nonzero hook exit. Claude Code surfaces the failure via the PostToolUse machinery; the next tool call's hook invocation will emit a fresh audit entry. The tradeoff per RESEARCH §1.6 "option 1 for hook":
- drop-the-entry (this plan): losing one audit event is fine; the next PostToolUse will capture the subsequent tool call
- unlocked-fallback (supervisor): losing a supervisor audit event is the entire record of a scheduled run — not recoverable

Idiom choice: FD-form (same as Plan 05-04). The hook's single-echo body makes command-form equally viable, but using the same FD-form in both writers keeps the grep-verifiable "one idiom, one mutex" property crisp for future maintainers.
