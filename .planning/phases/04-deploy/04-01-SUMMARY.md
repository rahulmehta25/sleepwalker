## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Initial bundleMtime test pinned only file mtimes; directory mtime sat at "now"**

- **Found during:** Task 2 GREEN phase, first full-suite run.
- **Issue:** The first draft of `"bundleMtime across files returns max across dir contents"` called `fs.utimesSync` on three children with staggered mtimes in 2025-01-01, then asserted that the newest child's mtime equaled the returned max. The implementation correctly includes `dirStat.mtimeMs` in its `Math.max`, and on macOS the directory's mtime is "now" (≈ 2026-04-20), so the returned max was the directory's mtime, not the newest child's. The test failed with `expected 1776645537975.861 to be 1735707603000`.
- **Fix:** Added `fs.utimesSync(dir, baseTime, baseTime)` as the FINAL call in the test (after all child utimesSync calls), because child utimesSync on macOS bumps parent dir mtime forward. This keeps the test exercising the "max across children" invariant cleanly while staying faithful to the `Math.max(dirStat.mtimeMs, ...children)` contract.
- **Files modified:** `dashboard/tests/deploy-state.test.ts` (inline fix in the same test block)
- **Commit:** `e3526c1` (bundled into the GREEN-phase feat commit; the Rule 1 fix was identified and corrected before the single commit was made, not a separate fix-commit).

No Rule 2 or Rule 3 auto-fixes triggered during this plan.

## Architectural Notes

- The state-file path convention `<runtime>-<slug>.state.json` is **slash-free** by design (04-RESEARCH.md line 192) so it mirrors the launchd label (`com.sleepwalker.codex.daily-brief`) minus the `com.sleepwalker.` prefix. This gives grep-ability on state files and labels with the same slug identifier. A nested-path scheme (`codex/daily-brief.state.json`) would have required Runtime-union → dir-map lookups that aren't needed here.
- `void fs; void crypto;` in Task 1's scaffold was a conscious choice to keep the file exports stable across the two commits. Removed in Task 2 when every import is actively consumed by the filled bodies.
- The `computeStatus` "drift before disabled" ordering matters: a drifted + disabled routine shows **drift** (the alert state), not **disabled** (the quiet state). Documented inline and in the decisions frontmatter.
- `enabled?: boolean` is optional. Wave 0 callers (tests, forthcoming Server Actions) can pass `computeStatus({runtime, slug, bundleDir})` without a Runtime.enabled lookup while the enable/disable flag is still being wired in Plan 04-04.
- No interfaces in Phase 4 contracts were widened or narrowed — the exact shapes from 04-RESEARCH.md §Deploy State Machine Design (lines 157-190) are honored verbatim.
