    **deployRoutine({runtime, slug})** algorithm:
    1. readBundle(runtime, slug) -> if null, return `{ok:false, error:"Bundle not found", failedStep:"planning"}` (no state file written — nothing to roll back)
    2. Construct baseState = `{fleet:toFleetKey(runtime,slug), runtime, slug, startedAt: new Date().toISOString(), steps: {}, phase: {kind:"running", step:"planning", stepStartedAt:Date.now()}}`
    3. writeDeployState(baseState) — "planning" row visible to poll
    4. Defensive double-deploy guard: if a PRIOR state exists with phase.kind==="running" AND Date.now()-new Date(prior.startedAt).getTime() < 60_000 -> return `{ok:false, error:"deploy already in progress"}` (per 04-RESEARCH.md Open Question #1)
    5. Call adapter.healthCheck(); if !available -> runRollback("planning", reason), return
    6. Transition to "writing": update steps.planning.completedAt+elapsedMs; write `{phase: {kind:"running", step:"writing", stepStartedAt:now}}`. Call adapter.deploy(bundle). On failure -> runRollback("writing", error), return. Capture `artifact` + `warning`.
    7. Transition to "loading": same pattern. For codex/gemini, run `launchctl print gui/$UID/<label>` via execFile with 3-retry 100ms backoff per Pitfall #2. For claude-desktop, assert SKILL.md exists. For claude-routines, no-op. On failure -> runRollback("loading", error), return.
    8. Transition to "verified": write `{phase: {kind:"succeeded"}, verifiedAt: Date.now(), artifact, warning}`.
    9. Return `{ok:true, state}`.

    **runRollback(failedStep, error)**:
    - rollbackActions: Array<{action, ok, error?}>
    - Try `await Promise.race([adapter.undeploy(bundle), timeout(10_000, "undeploy timed out after 10s")])`; push result
    - Try deleteDeployState; push result
