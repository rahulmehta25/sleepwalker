```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

permissions:
  contents: read

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  verify:
    name: Verify on macOS
    runs-on: macos-14
    steps:
      - name: Checkout (full history for COMP-02 baseline)
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install flock (required by supervisor + hooks)
        run: brew install discoteq/discoteq/flock

      - name: Install jq (v0.1 prereq)
        run: |
          if ! command -v jq >/dev/null 2>&1; then
            brew install jq
          else
            echo "jq already installed: $(jq --version)"
          fi

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 10

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
          cache-dependency-path: dashboard/pnpm-lock.yaml

      - name: Install dependencies
        working-directory: dashboard
        run: pnpm install --frozen-lockfile

      - name: Typecheck (dashboard)
        working-directory: dashboard
        run: pnpm run typecheck

      - name: Dashboard tests (Vitest)
        working-directory: dashboard
        run: pnpm test --run

      - name: Hook tests (PreToolUse / PostToolUse)
        run: bash hooks/tests/run-tests.sh

      - name: Supervisor tests
        run: bash hooks/tests/supervisor-tests.sh

      - name: COMP-01 v0.1 backward-compat integration
        run: bash tests/compat/v01-routines.sh

      - name: COMP-02 frozen-surface gate
        run: bash tests/compat/frozen-surface.sh
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR from fork → CI runner | Untrusted code executes on macos-14 runner with `contents: read` permission only |
| CI runner → repo | Runner cannot push, cannot merge, cannot comment (no `pull-requests: write`) |
| CI runner → GitHub API | Restricted to `contents: read`; cannot access secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06-01 | E (Elevation of privilege) | Malicious PR executing on CI | mitigate | `permissions: contents: read` — least privilege. Forks cannot access secrets (GitHub Actions default for untrusted PRs). No `pull-requests: write` scope. |
| T-06-06-02 | I (Information disclosure) | CI logs leaking env vars | mitigate | No `env:` blocks with secrets; no `echo $GITHUB_TOKEN`; workflow invokes scripts that run on isolated $TEST_HOME via mktemp (per COMP-01 Plan 06-04 design). |
| T-06-06-03 | T (Tampering) | Third-party action compromise | mitigate | Only pinned-version first-party actions used: `actions/checkout@v4`, `actions/setup-node@v4`, `pnpm/action-setup@v4`. No random OSS-author actions. Pin-to-SHA upgrade is a v0.3 hardening. |
| T-06-06-04 | D (Denial of service) | Runaway test hanging CI for 6 hours | mitigate | GitHub Actions default job timeout is 6 hours; steps like `pnpm test --run` exit deterministically (Vitest exits after all tests run). Bash scripts have internal timeouts where relevant (flock -w 5 in supervisor). Concurrency `cancel-in-progress: true` cancels superseded runs. |
| T-06-06-05 | I (Information disclosure) | COMP-02 baseline SHA visible in logs | accept | 998455b is a public git commit; exposing it in CI logs is expected. No secrets involved. |
| T-06-06-06 | T (Tampering) | Workflow file modification by contributor | accept | GitHub's default behavior: PRs from forks cannot modify workflows until merged to main (workflow-file protection). Internal contributors' PRs that modify ci.yml are visible in PR review. |
</threat_model>
